# Verilog HDL 복습노트 02 - wire, reg, 4-state

## 4-state

| 값 | 의미 |
| --- | --- |
| `0` | logic low |
| `1` | logic high |
| `x` | 값이 정해지지 않았거나 충돌한 상태 |
| `z` | high-impedance, 선이 떠 있는 상태 |

## `wire`와 `reg`

| 항목 | `wire` | `reg` |
| --- | --- | --- |
| 기본 성격 | 연결선, net | 절차 블록에서 쓰는 변수 |
| 주 사용 위치 | `assign`, 포트 연결 | `always`, `initial` 내부 |
| 값 유지 방식 | 다른 곳이 계속 drive해야 한다 | 마지막 할당값을 유지 |
| 주의점 | drive가 없으면 `z`나 미정 상태를 볼 수 있다 | 이름이 `reg`여도 실제 HW register와 1:1 의미는 아니다 |

## `assign`과 procedural assignment

| 형태 | 용도 | 보통 같이 보는 것 |
| --- | --- | --- |
| `assign y = a & b;` | 조합논리 연결 | `wire` |
| `always @(*) y = a & b;` | 조합논리 절차 기술 | `reg` 또는 `logic` |
| `always @(posedge clk) q <= d;` | 순차논리 상태 저장 | `reg` 또는 `logic` |

## 자주 헷갈리는 점

| 오해 | 정리 |
| --- | --- |
| `reg`는 무조건 플립플롭이다. | 아니다. `always @(*)` 안에서 쓰면 조합논리 변수다. |
| 출력은 항상 `reg`가 좋다. | 어떻게 구동하는지에 따라 달라진다. |
| `x`, `z`는 실제 회로에서 중요하지 않다. | 시뮬레이션 디버깅에서 매우 중요하다. |

## 같이 기억할 연산자

| 연산자 | 의미 |
| --- | --- |
| `~` | bitwise NOT |
| `!` | logical NOT |
| `|` | bitwise OR |
| `||` | logical OR |
