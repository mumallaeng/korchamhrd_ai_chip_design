# Verilog HDL 복습노트 03 - Basys3와 XDC

## 한 줄 요약

XDC는 Verilog 문법이 아니라, 설계 포트를 실제 FPGA 핀과 연결하는 제약조건 파일이다.

## XDC가 하는 일

| 항목 | 내용 |
| --- | --- |
| 핀 연결 | `get_ports`로 RTL 포트를 실제 핀 번호와 연결 |
| I/O standard | 보드 전압 표준 지정 |
| 클럭 제약 | 필요하면 주파수/타이밍 기준 설정 |

## Basys3에서 주로 보는 것

| 확인 대상 | 예시 |
| --- | --- |
| 스위치 | `sw[0]`, `sw[1]` |
| LED | `led[0]`, `led[1]` |
| 버튼 | `btnU`, `btnD`, `btnL`, `btnR`, `btnC` |
| FND | `fnd_data`, `fnd_com` |

## 오류가 나는 대표 이유

| 원인 | 설명 |
| --- | --- |
| 포트 이름 불일치 | RTL 포트 이름과 XDC 이름이 다름 |
| 핀 번호 오류 | 보드 문서와 다른 핀을 사용함 |
| I/O standard 누락 | 전기적 조건이 빠짐 |
| active-low 특성 미확인 | FND 같은 출력이 반대로 보임 |

## 실습 체크리스트

| 순서 | 체크 |
| --- | --- |
| 1 | Basys3 문서에서 핀 정보를 찾는다. |
| 2 | RTL 포트 이름을 먼저 확정한다. |
| 3 | XDC에 `PACKAGE_PIN`, `IOSTANDARD`를 맞춘다. |
| 4 | Implementation에서 제약 오류가 없는지 확인한다. |
