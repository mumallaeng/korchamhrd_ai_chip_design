## 이 장의 핵심

- Verilog의 기본 신호 표현은 `net`과 `variable` 관점으로 구분해 이해해야 한다.
- `wire`, `reg`, 벡터, 상수 표기, part-select, concatenation은 이후 모든 코딩의 기초다.
- 연산자는 소프트웨어와 비슷해 보여도 하드웨어 의미로 읽어야 한다.
- `x`, `z` 같은 4-state 값은 시뮬레이션과 디버깅에서 매우 중요하다.

## 1. `wire`와 `reg`

Verilog 초반의 가장 흔한 혼동점이다.

- `wire`: 연결선 관점
- `reg`: 절차 블록 안에서 값을 보관하는 변수 관점

초보자에게는 "wire는 assign으로, reg는 always/initial로"라는 규칙이 가장 실용적이다.

### 문법만 먼저 보기

```verilog
wire a;
reg  b;

assign a = expr;

always @(*) begin
    b = expr;
end
```

### 예제

```verilog
module wire_reg_example (
    input  a,
    input  b,
    output y_and,
    output reg y_or
);
    assign y_and = a & b;

    always @(*) begin
        y_or = a | b;
    end
endmodule
```

### 해석

- `y_and`는 `assign`으로 계속 구동되므로 net 관점이다.
- `y_or`는 `always` 안에서 대입되므로 `reg`로 선언했다.
- `reg`가 반드시 플립플롭을 뜻하는 것은 아니다. 조합논리 `always @(*)` 안의 `reg`도 가능하다.

## 2. 4-state 값과 디버깅 감각

Verilog는 `0`, `1`만 쓰지 않는다.

- `0`
- `1`
- `x`: 미정 또는 충돌
- `z`: high impedance

### 형식만 먼저 보기

```verilog
1'b0
1'b1
1'bx
1'bz
```

### 예제

```verilog
module four_state_demo;
    reg a, b;

    initial begin
        a = 1'bx;
        b = 1'b1;
        #10;
        a = 1'bz;
        #10;
        $finish;
    end
endmodule
```

### 해석

- `x`가 전파되면 결과도 `x`가 되기 쉽다.
- `z`는 연결이 떠 있는 상태라 조합 결과에서 종종 `x`로 해석된다.
- Day 2 강의에서 맨 오른쪽 carry-in을 반드시 `0`으로 두라고 한 이유도 이 디버깅 감각과 연결된다.

## 3. 벡터와 비트 선택

실제 회로는 1비트보다 다비트 버스를 더 많이 쓴다.

### 문법만 먼저 보기

```verilog
input  [7:0] a;
wire   [3:0] b;
reg    [15:0] data;

a[0]
a[7:4]
```

### 예제

```verilog
module vector_demo (
    input  [7:0] a,
    output [3:0] low_nibble,
    output [3:0] high_nibble
);
    assign low_nibble  = a[3:0];
    assign high_nibble = a[7:4];
endmodule
```

### 해석

- `[7:0]`는 8비트 벡터다.
- `a[3:0]`는 하위 4비트 part-select다.
- 8비트 입력을 4비트 두 조각으로 나누는 방식은 이후 `digit_splitter`나 상위/하위 nibble 처리에서 자주 나온다.

## 4. 수치 표현

Verilog 상수는 비트 수와 진법을 같이 적는 습관이 중요하다.

### 문법만 먼저 보기

```verilog
4'b1010
8'hC0
8'd255
16'o777
```

### 예제

```verilog
module literal_demo;
    reg [7:0] a;
    reg [7:0] b;

    initial begin
        a = 8'd10;
        b = 8'h0A;
    end
endmodule
```

### 해석

- `8'd10`과 `8'h0A`는 같은 값을 다른 진법으로 쓴 것이다.
- 비트 폭을 명시하면 잘림이나 sign-extension 문제를 줄일 수 있다.
- FND 디코더에서 `8'hc0` 같은 세그먼트 패턴이 자주 등장한다.

## 5. 주요 연산자

자주 쓰는 연산자는 다음과 같이 분류해서 기억하는 편이 좋다.

- bitwise: `~`, `&`, `|`, `^`
- logical: `!`, `&&`, `||`
- 산술: `+`, `-`, `*`, `/`, `%`
- 비교: `==`, `!=`, `<`, `>`
- shift: `<<`, `>>`
- 결합: `{}`, `{{}}`

### 문법만 먼저 보기

```verilog
y0 = a & b;
y1 = a | b;
y2 = a ^ b;
y3 = {a, b};
y4 = data[7:0] << 1;
```

### 예제

```verilog
module operator_demo (
    input  [3:0] a,
    input  [3:0] b,
    output [4:0] sum,
    output [7:0] packed
);
    assign sum    = a + b;
    assign packed = {a, b};
endmodule
```

### 해석

- `sum`은 자리올림 때문에 5비트로 받는 편이 안전하다.
- `{a, b}`는 두 벡터를 붙여 8비트로 만든다.
- 하드웨어 설계에서는 같은 기호라도 "값 계산"뿐 아니라 "비트 구조 형성" 의미가 있다.

## 6. `parameter`와 재사용

비슷한 모듈을 폭만 바꿔서 다시 쓰고 싶을 때 쓴다.

### 문법만 먼저 보기

```verilog
module demo #(
    parameter WIDTH = 8
) (
    input  [WIDTH-1:0] a,
    output [WIDTH-1:0] y
);
endmodule
```

### 예제

```verilog
module passthrough #(
    parameter WIDTH = 8
) (
    input  [WIDTH-1:0] a,
    output [WIDTH-1:0] y
);
    assign y = a;
endmodule
```

### 해석

- 같은 코드를 4비트, 8비트, 16비트용으로 재사용할 수 있다.
- 6장의 구조적 모델링과 바로 이어지는 개념이다.

## 자주 하는 실수

- `always`에서 대입하는 신호를 `wire`로 선언한다.
- 벡터 폭을 맞추지 않는다.
- `!`와 `~`, `|`와 `||`의 의미를 혼동한다.
- `x`, `z`가 섞인 파형을 보고도 원인을 추적하지 않는다.
