## 이 장의 핵심

- 게이트 수준 모델링은 논리 게이트 프리미티브를 직접 연결하는 방식이다.
- 회로 구조를 아주 직관적으로 볼 수 있지만, 복잡해질수록 유지보수성이 떨어진다.
- 지연 모델을 붙여 시뮬레이션 행동을 더 세밀하게 볼 수 있다.
- 초반 학습용으로는 좋지만, 실무 RTL은 보통 더 상위 수준 표현을 함께 사용한다.

## 1. 게이트 프리미티브

Verilog에는 기본 게이트를 직접 부르는 문법이 있다.

### 문법만 먼저 보기

```verilog
and  g0 (y, a, b);
or   g1 (y, a, b);
not  g2 (y, a);
xor  g3 (y, a, b);
nand g4 (y, a, b);
```

### 예제

```verilog
module gates_primitive (
    input  a,
    input  b,
    output y_and,
    output y_or,
    output y_xor
);
    and u0 (y_and, a, b);
    or  u1 (y_or,  a, b);
    xor u2 (y_xor, a, b);
endmodule
```

### 해석

- 프리미티브도 인스턴스처럼 이름을 붙일 수 있다.
- 첫 번째 인자가 출력이고, 나머지가 입력이다.
- 게이트 상징도와 매우 비슷해서 입문 단계에서 구조를 이해하기 좋다.

## 2. 게이트 수준 모델링의 특징

장점:

- 회로 구조가 직접 보인다.
- 논리식과 게이트 연결 관계가 분명하다.
- 간단한 회로 교육용으로 적합하다.

단점:

- 큰 회로는 코드가 길어진다.
- 수정이 번거롭다.
- 가독성이 빠르게 나빠진다.

그래서 게이트 수준 모델링은 `회로 본질을 배우는 단계`에서 특히 유용하다.

## 3. 게이트 지연

프리미티브에는 지연을 줄 수 있다.

### 문법만 먼저 보기

```verilog
and #(3) u0 (y, a, b);
or  #(2, 3) u1 (y, a, b);
```

### 예제

```verilog
module delayed_gate (
    input  a,
    input  b,
    output y
);
    and #(5) u0 (y, a, b);
endmodule
```

### 해석

- 지연은 시뮬레이션에서 출력 변화 시점을 늦춰 본다.
- 실제 FPGA 하드웨어 지연을 정확히 뜻하는 것은 아니다.
- 입문 단계에서는 "지연이라는 모델을 줄 수 있다" 정도를 아는 것으로 충분하다.

## 4. 데이터흐름 표현과 비교

같은 AND 게이트를 더 간단히 쓸 수도 있다.

### 문법만 먼저 보기

게이트 수준:

```verilog
and u0 (y, a, b);
```

데이터흐름 수준:

```verilog
assign y = a & b;
```

### 예제

```verilog
module compare_style (
    input  a,
    input  b,
    output y0,
    output y1
);
    and u0 (y0, a, b);
    assign y1 = a & b;
endmodule
```

### 해석

- 논리 의미는 같다.
- 아래쪽 `assign`이 더 짧고 관리가 쉽다.
- 그래서 실제 RTL 설계는 대개 `assign` 또는 `always`를 더 많이 쓴다.

## 5. 반가산기 예제

게이트 수준 모델링의 대표적인 입문 예제다.

### 문법만 먼저 보기

```verilog
module half_adder (...);
    xor (...);
    and (...);
endmodule
```

### 예제

```verilog
module half_adder_gate (
    input  a,
    input  b,
    output sum,
    output carry
);
    xor u0 (sum,   a, b);
    and u1 (carry, a, b);
endmodule
```

### 해석

- 합은 XOR
- 자리올림은 AND
- 진리표와 게이트 심볼이 바로 대응된다.

## 6. 전가산기 예제

조금 더 큰 회로는 하위 블록을 구조적으로 연결한다.

### 문법만 먼저 보기

```verilog
module full_adder (...);
    wire s1, c1, c2;
    half_adder u0 (...);
    half_adder u1 (...);
    or         u2 (...);
endmodule
```

### 예제

```verilog
module full_adder_gate (
    input  a,
    input  b,
    input  cin,
    output sum,
    output cout
);
    wire s1, c1, c2;

    xor u0 (s1,   a,   b);
    xor u1 (sum,  s1,  cin);
    and u2 (c1,   a,   b);
    and u3 (c2,   s1,  cin);
    or  u4 (cout, c1,  c2);
endmodule
```

### 해석

- 전가산기는 내부 중간 신호가 필요하다.
- `wire s1, c1, c2`가 바로 중간 배선이다.
- Day 2의 전가산기 설명과 본질적으로 같은 구조다.

## 7. 게이트 수준 모델링을 언제 쓰는가

- 간단한 교육용 예제
- 아주 작은 조합 회로
- 합성 결과와 논리 구조를 대응시키며 배울 때
- 라이브러리 셀 수준에 가까운 저수준 설명이 필요할 때

하지만 설계가 커지면 보통 다음으로 넘어간다.

- `assign`
- `always @(*)`
- 구조적 상위 모듈 조립

## 자주 하는 실수

- 프리미티브 포트 순서를 헷갈린다.
- 게이트 지연을 실제 칩 지연과 동일시한다.
- 큰 회로까지 게이트 프리미티브만으로 밀어붙인다.
- 중간 `wire` 선언을 빠뜨린다.

## 복습 질문

1. 게이트 프리미티브 방식의 장점은 무엇인가

3. 게이트 지연은 왜 주로 시뮬레이션 관점에서 보아야 하는가
4. 반가산기와 전가산기를 게이트 수준으로 표현하면 어떤 중간 신호가 필요한가
