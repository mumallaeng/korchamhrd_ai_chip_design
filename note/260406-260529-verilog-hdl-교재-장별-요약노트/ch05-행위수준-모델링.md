## 이 장의 핵심

- 행위수준 모델링은 회로의 동작을 `always`, `initial`, `if`, `case`, `for` 등으로 기술하는 방식이다.
- 조합논리와 순차논리는 `민감도 목록`과 `할당 방식`이 다르다.
- `always @(*)`와 `always @(posedge clk ...)`의 차이는 이후 설계 품질을 좌우한다.
- 블로킹과 논블로킹, 래치 방지, 기본값 처리 규칙은 초반에 확실히 잡아야 한다.

## 세부 목차

1. `always`와 `initial`
2. 조합논리 모델링
3. 순차논리 모델링
4. `if`, `case`, `for`
5. blocking / non-blocking

## 한 줄 요약

5장은 Verilog를 "회로를 설명하는 절차적 언어"처럼 쓰는 장이다. 문법은 절차형처럼 보여도, 결국 하드웨어로 읽어야 한다는 점이 핵심이다.

## 현재 수업과 연결

- Day 3~5의 `mux`, `BCD`, `counter`, `control_unit`, `tick_counter`와 직접 연결된다.
- 강의에서 반복해서 나온 `always @(*)`, `default`, `latch`, `posedge`, `<=`가 모두 이 장의 핵심이다.

## 1. `always`와 `initial`

`always`는 반복적으로 동작하는 procedural block이고, `initial`은 시뮬레이션 시작 시 한 번 실행되는 block이다.

### 문법만 먼저 보기

```verilog
always @(...) begin
    ...
end

initial begin
    ...
end
```

### 예제

```verilog
module proc_demo;
    reg clk;
    reg a;

    initial begin
        clk = 1'b0;
        a   = 1'b0;
    end

    always #5 clk = ~clk;
endmodule
```

### 해석

- `initial`은 테스트벤치에서 초기화와 자극 생성에 자주 쓴다.
- `always #5 clk = ~clk;`는 반복 동작의 대표 예다.
- 합성 대상 RTL에서 `initial` 사용은 제한적이므로 보통 검증 코드에서 먼저 익힌다.

## 2. 조합논리 모델링

조합논리는 현재 입력만으로 결과가 정해진다.

### 문법만 먼저 보기

```verilog
always @(*) begin
    y = ...;
end
```

### 예제

```verilog
module mux_4x1 (
    input  [3:0] input0,
    input  [3:0] input1,
    input  [3:0] input2,
    input  [3:0] input3,
    input  [1:0] sel,
    output reg [3:0] out_mux
);
    always @(*) begin
        case (sel)
            2'b00: out_mux = input0;
            2'b01: out_mux = input1;
            2'b10: out_mux = input2;
            2'b11: out_mux = input3;
            default: out_mux = 4'b0000;
        endcase
    end
endmodule
```

### 해석

- `always @(*)`는 조합논리의 기본 틀이다.
- 강의에서처럼 sensitivity list를 모두 넣는 대신 `*`를 쓰면 입력 변화에 따라 자동으로 재평가된다.
- `default`를 넣는 이유는 래치 방지와 안전한 기본값 때문이다.

## 3. 순차논리 모델링

순차논리는 클럭이나 이벤트 엣지에 반응해 상태가 바뀐다.

### 문법만 먼저 보기

```verilog
always @(posedge clk or posedge rst) begin
    if (rst)
        q <= 0;
    else
        q <= d;
end
```

### 예제

```verilog
module counter_4 (
    input        clk,
    input        rst,
    output reg [1:0] q
);
    always @(posedge clk or posedge rst) begin
        if (rst)
            q <= 2'b00;
        else
            q <= q + 1'b1;
    end
endmodule
```

### 해석

- `posedge clk`에서만 상태가 갱신된다.
- `rst`는 비동기 리셋 예시다.
- Day 3~5 수업에서 본 `counter_4`, `clk divider`, `tick counter`가 이 틀에서 나왔다.

## 4. `if`, `case`, `for`

행위수준 모델링은 절차 구조를 활용한다.

### 문법만 먼저 보기

```verilog
if (...) begin
    ...
end else begin
    ...
end

case (sel)
    ...
    default: ...
endcase

for (i = 0; i < N; i = i + 1) begin
    ...
end
```

### 예제

```verilog
always @(*) begin
    if (en)
        y = a;
    else
        y = b;
end
```

### 해석

- `if`는 우선순위가 있는 선택 구조로 읽히는 경우가 많다.
- `case`는 명시적인 경우 분기다.
- `for`는 절차형 반복이며, 합성에서는 범위가 명확해야 한다.

## 5. blocking과 non-blocking

강의에서 가장 반복해서 강조한 규칙이다.

- 조합논리 `always @(*)`에서는 보통 `=`
- 순차논리 `always @(posedge clk ...)`에서는 보통 `<=`

### 문법만 먼저 보기

```verilog
// combinational
y = a & b;

// sequential
q <= d;
```

### 예제

```verilog
always @(*) begin
    next_state = current_state;
end

always @(posedge clk or posedge rst) begin
    if (rst)
        current_state <= IDLE;
    else
        current_state <= next_state;
end
```

### 해석

- 블로킹은 절차 순서대로 바로 반영되는 느낌이다.
- 논블로킹은 클럭 이벤트 시점의 상태 갱신을 모델링하는 데 적합하다.
- 한 블록 안에서 둘을 섞어 쓰지 않는 습관이 중요하다.

## 6. 래치 방지

조합논리 `always`에서 모든 경우를 다 다루지 않으면 의도치 않은 래치가 생길 수 있다.

### 형식만 먼저 보기

```verilog
always @(*) begin
    y = default_value;
    if (...) begin
        y = ...;
    end
end
```

### 예제

```verilog
always @(*) begin
    out_reg = 4'b0000;
    case (sel)
        2'b00: out_reg = input0;
        2'b01: out_reg = input1;
        2'b10: out_reg = input2;
        2'b11: out_reg = input3;
    endcase
end
```

### 해석

- 기본값을 먼저 주면 경우를 빠뜨렸을 때도 이전 상태 유지 래치를 막기 쉽다.
- 강의에서 `full case 처리`를 강조한 이유가 여기 있다.

## 7. 테스트벤치의 행위수준 코드

행위수준 모델링은 테스트벤치에서 특히 강력하다.

### 문법만 먼저 보기

```verilog
initial begin
    a = 0;
    #10 a = 1;
    #10 $finish;
end
```

### 예제

```verilog
module tb_counter;
    reg clk;
    reg rst;
    wire [1:0] q;

    counter_4 dut (.clk(clk), .rst(rst), .q(q));

    initial clk = 1'b0;
    always #5 clk = ~clk;

    initial begin
        rst = 1'b1;
        #12 rst = 1'b0;
        #50 $finish;
    end
endmodule
```

### 해석

- 클럭 생성, 리셋 인가, 종료 제어를 procedural하게 쓸 수 있다.
- 이런 코드는 검증용이며 FPGA 내부 하드웨어가 직접 되는 것은 아니다.

## 자주 하는 실수

- 조합논리에서 sensitivity list를 빼먹는다.
- 조합논리에서 `<=`를 습관적으로 쓴다.
- 순차논리에서 `=`를 써서 파형 해석을 꼬이게 만든다.
- `case`에 `default`를 두지 않는다.
- 테스트벤치 문법과 합성 RTL 문법을 구분하지 않는다.
