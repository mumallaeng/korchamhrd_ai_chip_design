## 이 장의 핵심

- 함수와 태스크는 반복되는 절차를 묶어 코드 재사용성을 높이는 수단이다.
- 함수는 값을 계산해 반환하는 데 적합하고, 태스크는 여러 입출력과 절차를 묶는 데 적합하다.
- 함수는 기본적으로 `시간 지연 없이 조합적 계산`에 더 가깝고, 태스크는 테스트벤치 제어에 특히 유용하다.
- `Verilog function`은 C처럼 `return`을 쓰지 않고, `함수 이름 자체에 값을 대입`해 반환한다.
- `task`는 반환값이 없고, 필요하면 `output`이나 `inout`으로 결과를 전달한다.
- 코드를 줄이는 목적보다 `의미 단위로 묶어 읽기 좋게 만드는 목적`이 더 중요하다.

## 현재 수업과 연결

- 현재 수업 예제는 아직 큰 함수/태스크 사용 비중이 크지 않지만, 테스트벤치 검증과 반복 자극 생성에서 바로 쓸 수 있다.
- 이후 `fsm_led`, 카운터 검증, 반복적인 파형 체크를 정리할 때 태스크가 특히 유용하다.

## 1. 함수와 태스크가 필요한 이유

코드가 길어지면 같은 계산이나 같은 자극 시퀀스를 여러 번 쓰게 된다.

- 함수: 하나의 계산 논리를 이름으로 묶기 좋다.
- 태스크: 여러 단계 절차를 한 묶음으로 재사용하기 좋다.

즉 둘 다 `중복 제거`가 목적이지만, 쓰임새는 다르다.

## 2. 함수

함수는 계산 결과를 반환하는 데 초점을 둔다.

### 문법만 먼저 보기

```verilog
function [WIDTH-1:0] func_name;
    input ...;
    begin
        func_name = ...;
    end
endfunction
```

### 예제

```verilog
module parity_example (
    input  [7:0] data,
    output       parity
);
    function calc_parity;
        input [7:0] x;
        begin
            calc_parity = ^x;
        end
    endfunction

    assign parity = calc_parity(data);
endmodule
```

### 해석

- 함수 이름 자체에 반환값을 대입한다.
- `calc_parity(data)`처럼 표현식처럼 쓸 수 있다.
- 조합 계산을 묶는 용도로 특히 자연스럽다.

## 2.1 Verilog 함수에는 `return`이 없나

지금 수업 문맥의 `Verilog HDL` 기준으로는, C처럼 `return expr;`를 기본 문법으로 생각하면 안 된다.

### 문법만 먼저 보기

```verilog
function [3:0] func_name;
    input [3:0] x;
    begin
        func_name = x + 1'b1;
    end
endfunction
```

### 예제

```verilog
function [3:0] add1;
    input [3:0] x;
    begin
        add1 = x + 1'b1;
    end
endfunction

assign y = add1(a);
```

### 해석

- `add1 = ...;` 이 문장이 곧 반환값 지정이다.
- 함수 이름이 반환용 임시 변수처럼 동작한다고 생각하면 된다.
- 그래서 `Verilog function`은 `return 값을 함수명에 대입해서 끝낸다`고 기억하는 편이 가장 정확하다.
- `SystemVerilog`에서는 `return` 문을 볼 수 있지만, 현재 수업 문맥의 `Verilog HDL`은 이 방식으로 이해하는 편이 맞다.

## 2.2 왜 이렇게 이해하는 편이 좋은가

- `function`은 수식 안에 들어갈 수 있는 계산 블록으로 읽는 편이 자연스럽다.
- 반환값이 하나라는 점이 분명하다.
- `task`와 비교할 때 차이가 선명해진다.

## 3. 태스크

태스크는 여러 문장과 여러 입출력을 묶는 데 적합하다.

### 문법만 먼저 보기

```verilog
task task_name;
    input  ...;
    output ...;
    begin
        ...
    end
endtask
```

### 예제

```verilog
module tb_task_demo;
    reg clk;
    reg rst;
    reg a;

    task apply_reset;
        begin
            rst = 1'b1;
            #12 rst = 1'b0;
        end
    endtask

    initial begin
        clk = 1'b0;
        rst = 1'b0;
        a   = 1'b0;

        apply_reset();
        #20 a = 1'b1;
        #20 $finish;
    end

    always #5 clk = ~clk;
endmodule
```

### 해석

- 리셋 시퀀스를 태스크 하나로 묶었다.
- `#12`처럼 시간 제어가 있어도 된다.
- 테스트벤치에서 자주 쓰는 이유가 바로 여기에 있다.

## 3.1 태스크는 어떻게 결과를 밖으로 내보내나

태스크는 함수처럼 "반환값 1개"라는 개념이 기본이 아니다.

### 문법만 먼저 보기

```verilog
task task_name;
    input  a;
    input  b;
    output y;
    begin
        y = a + b;
    end
endtask
```

### 예제

```verilog
task do_add;
    input  [3:0] a;
    input  [3:0] b;
    output [4:0] s;
    begin
        s = a + b;
    end
endtask
```

### 해석

- 태스크는 `output` 또는 `inout`을 통해 바깥으로 값을 전달한다.
- 그래서 `return`이 없는 절차 묶음으로 이해하는 편이 맞다.
- 테스트벤치에서 여러 신호를 동시에 만지는 절차에 특히 잘 맞는다.

## 4. 함수와 태스크의 차이

핵심 비교는 아래처럼 정리하면 된다.

| 항목 | 함수 | 태스크 |
| --- | --- | --- |
| 목적 | 값 계산 | 절차 묶음 |
| 반환 | 하나의 반환값 | 여러 output 가능 |
| 사용 형태 | 표현식처럼 호출 | 문장처럼 호출 |
| 시간 제어 | 일반적으로 쓰지 않음 | 가능 |
| 반환 방식 | 함수명에 대입 | `output/inout` 사용 |
| 대표 위치 | RTL 조합 계산 | 테스트벤치 시퀀스 |

### 형식만 먼저 보기

함수 호출:

```verilog
y = func_name(a, b);
```

태스크 호출:

```verilog
task_name(a, b, c);
```

### 해석

- 함수는 수식 속 부품처럼 보인다.
- 태스크는 명령 블록을 호출하는 느낌에 가깝다.

## 5. 함수는 언제 좋은가

- 반복되는 조합 계산
- parity, encode, min/max 같은 계산
- 식을 짧고 의미 있게 만들고 싶을 때

### 예제

```verilog
function [3:0] bcd_clip;
    input [3:0] x;
    begin
        if (x > 4'd9)
            bcd_clip = 4'd9;
        else
            bcd_clip = x;
    end
endfunction
```

### 해석

- BCD 범위를 보정하는 계산을 함수로 묶었다.
- 이후 FND 관련 설계에서도 유사한 보조 계산을 깔끔하게 분리할 수 있다.

## 6. 태스크는 언제 좋은가

- 테스트벤치 자극 반복
- 상태 확인 절차 묶기
- 로그 출력 절차 표준화

### 예제

```verilog
task check_led;
    input [2:0] expected;
    begin
        if (led !== expected)
            $display("ERROR expected=%b actual=%b", expected, led);
    end
endtask
```

### 해석

- `fsm_led` 테스트벤치에서 상태별 LED 확인을 반복할 때 매우 유용하다.
- 같은 비교 코드를 여러 번 복붙할 필요가 없다.

## 7. RTL에서 쓸 때 주의점

- 함수는 조합 계산 중심으로 생각하는 편이 안전하다.
- 태스크는 합성 가능한 RTL에도 쓸 수 있지만, 초반에는 테스트벤치용으로 먼저 익히는 것이 덜 헷갈린다.
- 함수와 태스크가 코드를 숨기기만 하고 의미를 흐리면 오히려 나쁘다.

## 자주 하는 실수

- 함수와 태스크를 단순 문법 차이로만 외운다.
- 함수에서 C식 `return`을 기대한다.
- 함수에 절차 제어를 넣으려 한다.
- 태스크를 써 놓고 호출부보다 더 읽기 어렵게 만든다.
- 재사용 단위를 너무 작게 쪼개 오히려 가독성을 해친다.
