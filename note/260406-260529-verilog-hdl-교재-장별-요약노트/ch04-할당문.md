## 이 장의 핵심

- 할당문은 신호 사이의 관계를 Verilog 코드로 연결하는 가장 기본 수단이다.
- `assign`은 연속 할당이고, 조합논리 표현에 매우 적합하다.
- 절차형 할당은 `always`, `initial` 같은 procedural block 안에서 쓴다.
- 연속 할당과 절차형 할당을 섞어 읽지 않으면 이후 장이 훨씬 쉬워진다.

## 세부 목차

1. 연속 할당
2. 할당 지연
3. 절차형 할당과의 차이
4. `assign`으로 쓰기 좋은 회로
5. `assign`으로 쓰기 불편한 회로

## 한 줄 요약

4장은 "연결" 중심의 Verilog를 이해하는 장이다. 특히 `assign`은 코드 한 줄이 곧 회로 연결이라는 감각을 익히는 데 가장 중요하다.

## 현재 수업과 연결

- Day 1의 게이트 예제는 대부분 `assign`으로 표현되었다.
- Day 3 이후 `BCD`, 일부 `decoder`, 단순 조합 논리는 `assign`과 `always @(*)` 중 어느 쪽이 자연스러운지 계속 비교하게 된다.

## 1. 연속 할당

`assign`은 오른쪽 식이 바뀌면 왼쪽 net을 계속 다시 구동한다.

### 문법만 먼저 보기

```verilog
assign lhs = rhs_expression;
```

### 예제

```verilog
module simple_assign (
    input  a,
    input  b,
    output y
);
    assign y = a & b;
endmodule
```

### 해석

- `y`는 `a & b`와 항상 연결된다.
- 코드 순서대로 "한 번 실행"되는 개념이 아니다.
- 조합논리를 가장 간결하게 쓰는 방법 중 하나다.

## 2. 연속 할당으로 쓰기 좋은 경우

- 단순 논리식
- 단순 결합/분리
- 조건연산자 기반 mux
- 비교 결과 출력

### 문법만 먼저 보기

```verilog
assign y = sel ? a : b;
assign low = data[3:0];
assign eq  = (a == b);
```

### 예제

```verilog
module mux2_assign (
    input  a,
    input  b,
    input  sel,
    output y
);
    assign y = sel ? b : a;
endmodule
```

### 해석

- 2:1 mux를 `assign` 한 줄로 표현했다.
- if/case가 없어도 간단한 선택 회로는 충분히 표현 가능하다.

## 3. 할당 지연

연속 할당에도 지연을 줄 수 있다.

### 문법만 먼저 보기

```verilog
assign #5 y = a & b;
```

### 예제

```verilog
module delayed_assign (
    input  a,
    input  b,
    output y
);
    assign #3 y = a ^ b;
endmodule
```

### 해석

- 지연은 시뮬레이션 모델이다.
- 합성 대상 RTL에서는 보통 이런 표현에 의존하지 않는다.

## 4. 절차형 할당과의 차이

초반에 가장 많이 헷갈리는 포인트다.

### 문법만 먼저 보기

연속 할당:

```verilog
assign y = a & b;
```

절차형 할당:

```verilog
always @(*) begin
    y = a & b;
end
```

### 예제

```verilog
module compare_assign_proc (
    input  a,
    input  b,
    output y0,
    output reg y1
);
    assign y0 = a & b;

    always @(*) begin
        y1 = a & b;
    end
endmodule
```

### 해석

- `y0`는 net 연결
- `y1`는 procedural assignment 결과
- 둘 다 조합논리일 수 있지만, 코딩 규칙과 선언 방식이 다르다.

## 5. `assign`과 벡터

벡터 처리에도 매우 자주 쓴다.

### 문법만 먼저 보기

```verilog
assign upper = data[7:4];
assign lower = data[3:0];
assign both  = {a, b};
```

### 예제

```verilog
module vector_assign (
    input  [7:0] data,
    output [3:0] upper,
    output [3:0] lower
);
    assign upper = data[7:4];
    assign lower = data[3:0];
endmodule
```

### 해석

- slicing과 concatenation은 `assign`과 잘 어울린다.
- 버스 분리/결합은 실습 회로에서 계속 등장한다.

## 6. `assign`으로 불편해지는 순간

아래 상황에서는 `always`가 더 자연스럽다.

- 조건 분기가 많다
- `case`가 필요하다
- 기본값과 예외 처리를 분명히 하고 싶다
- 조합논리라도 사람이 읽기 쉽게 구조화하고 싶다

예를 들어 `BCD to 7-segment`처럼 경우의 수가 많으면 보통 `always @(*) + case`가 더 낫다.

## 7. `assign`과 tri-state 감각

고전 Verilog 문맥에서는 `z`를 이용한 tri-state 표현도 나온다.

### 문법만 먼저 보기

```verilog
assign data = en ? value : 1'bz;
```

### 예제

```verilog
module tri_state_demo (
    input  en,
    input  value,
    output bus
);
    assign bus = en ? value : 1'bz;
endmodule
```

### 해석

- 교육용 문법으로는 중요하다.
- 하지만 FPGA 내부 로직에서는 실제 tri-state가 제한적이므로 툴이 mux 구조로 바꾸는 경우가 많다.

## 자주 하는 실수

- `assign` 대상 신호를 `reg`로 선언한다.
- `assign`을 순차 실행처럼 읽는다.
- 지연이 실제 하드웨어 지연과 같다고 생각한다.
- 큰 case형 조합논리를 억지로 한 줄 `assign`으로 압축한다.
