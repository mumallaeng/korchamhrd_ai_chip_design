# 26-04-09 - 합성, 구조 확장, FND 시스템 정리

## 한눈에 보기

| 항목 | 내용 |
| --- | --- |
| 목표 | 조합논리 예제를 넘어 top, datapath, tick generator가 있는 구조로 확장한다 |
| 핵심 | `counter_10000` top, `project_datapath`, `tick_counter_project`, `clk_tick_gen_project`, `fnd_controller` |
| 기준 | 현재 저장소의 `counter_10000` 프로젝트는 수업본보다 정리된 버전이므로 구조 중심으로 읽는다 |

## 관련 코드 위치

| 파일 | 위치 | 의미 |
| --- | --- | --- |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000.v` | `3-98` | top module |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000_common.v` | `3-37` | datapath 연결 |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000_common.v` | `39-72` | up/down 10000 카운터 |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000_common.v` | `74-105` | tick generator |

## 전체 구조

```text
button 입력
-> control_unit
-> datapath
-> tick generator
-> 0~9999 counter
-> fnd_controller
-> FND 표시
```

## 이 날 핵심

| 항목 | 정리 |
| --- | --- |
| top module | 버튼, FND, 내부 제어 블록을 한곳에서 묶는다 |
| datapath | 실제 값이 흐르고 바뀌는 부분 |
| tick generator | 빠른 `clk`를 사람이 보기 좋은 enable pulse로 바꾼다 |
| counter | `9999 -> 0`, `0 -> 9999` wrap 동작을 만든다 |

## 코드에서 바로 읽을 것

| 위치 | 읽는 포인트 |
| --- | --- |
| `counter_10000.v:65-87` | top이 `control_unit`, `project_datapath`, `fnd_controller`를 묶는다 |
| `counter_10000_common.v:17-24` | datapath 안에서 tick과 counter를 연결한다 |
| `counter_10000_common.v:55-68` | mode에 따라 up/down 경로가 갈린다 |
| `counter_10000_common.v:95-103` | `o_tick`은 한 클럭짜리 pulse다 |

## 합성 관점에서 봐야 하는 것

| 질문 | 답 |
| --- | --- |
| 왜 top과 하위 모듈을 나누는가 | 기능 분리, 디버깅, 재사용이 쉬워진다 |
| 왜 `tick`을 별도 신호로 두는가 | 느린 clock 대신 enable pulse로 제어하기 쉽다 |
| 왜 FND를 마지막에 둬야 하는가 | 계산 결과와 표시를 분리할 수 있다 |
