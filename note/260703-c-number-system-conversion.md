## 8. C문법 조금 더 알아보기

8번 단원은 C 문법 자체를 조금 더 깊게 다룬다. 7장까지는 조건문, 반복문, 배열, 구조체처럼 프로그램 구조를 만드는 문법을 봤고, 8장부터는 값이 memory와 출력에서 어떻게 표현되는지까지 함께 본다.

진법 변환은 이후 bit 연산, register 값, 주소, endian을 이해하는 기초가 된다. FPGA, MCU, register map을 볼 때 10진수보다 2진수와 16진수 표현을 자주 사용하므로 자리수의 가중치를 먼저 익혀야 한다.

## 8.1 2진수와 10진수 변환

2진수는 각 자리의 값이 `2`의 거듭제곱으로 정해진다. 오른쪽 끝 bit부터 `1`, `2`, `4`, `8` 순서로 커지고, 왼쪽으로 갈수록 자리수 값이 2배씩 증가한다.

| 자리수 | 256 | 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 |
| :--- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 2의 거듭제곱 | `2^8` | `2^7` | `2^6` | `2^5` | `2^4` | `2^3` | `2^2` | `2^1` | `2^0` |

2진수에서 해당 자리의 bit가 `1`이면 그 자리수 값을 더하고, `0`이면 더하지 않는다.

## 2진수와 16진수 표현

2진수는 bit 단위의 상태를 그대로 보여주므로 하드웨어 동작을 이해할 때 정확하지만, 자리수가 길어지면 읽기 불편하다. 그래서 2진수를 `4bit`씩 묶어 16진수 한 자리로 표현한다.

`4bit`로 표현할 수 있는 경우의 수는 `0000`부터 `1111`까지 총 16개다. 따라서 16진수 한 자리는 `0`부터 `15`까지의 값을 표현할 수 있고, `10`부터 `15`는 `A`부터 `F`로 적는다.

| 10진수 | 2진수 4bit | 16진수 |
| ---: | :---: | :---: |
| 0 | `0000` | `0` |
| 1 | `0001` | `1` |
| 2 | `0010` | `2` |
| 3 | `0011` | `3` |
| 4 | `0100` | `4` |
| 5 | `0101` | `5` |
| 6 | `0110` | `6` |
| 7 | `0111` | `7` |
| 8 | `1000` | `8` |
| 9 | `1001` | `9` |
| 10 | `1010` | `A` |
| 11 | `1011` | `B` |
| 12 | `1100` | `C` |
| 13 | `1101` | `D` |
| 14 | `1110` | `E` |
| 15 | `1111` | `F` |

예를 들어 `10000001_2`는 오른쪽부터 `4bit`씩 묶으면 `1000 0001`이고, 이는 `0x81`로 표현된다. 같은 값이지만 2진수보다 짧고, bit 묶음도 보존되기 때문에 register 값이나 memory 주소를 볼 때 16진수를 자주 사용한다.

## 2진수와 8진수 표현

8진수는 2진수를 `3bit`씩 묶어 표현한다. `3bit`로 표현할 수 있는 경우의 수는 `000`부터 `111`까지 총 8개이므로, 8진수 한 자리는 `0`부터 `7`까지의 값을 가진다.

| 10진수 | 2진수 3bit | 8진수 |
| ---: | :---: | :---: |
| 0 | `000` | `0` |
| 1 | `001` | `1` |
| 2 | `010` | `2` |
| 3 | `011` | `3` |
| 4 | `100` | `4` |
| 5 | `101` | `5` |
| 6 | `110` | `6` |
| 7 | `111` | `7` |

8진수 표기는 앞에 `0`을 붙인다. 예를 들어 `1111_2`는 오른쪽부터 `3bit`씩 묶기 위해 왼쪽에 `0`을 채워 `001 111`로 보고, 이는 `017`로 표현된다.

```text
1111_2 = 001 111_2 = 017_8 = 15_10
```

앞에 `0`이 붙은 정수는 C에서 8진수 literal로 해석될 수 있으므로 주의해야 한다. 예를 들어 `10`은 10진수 `10`이지만, `010`은 8진수 `10`이므로 10진수 값은 `8`이다.

## 2진수에서 10진수로 변환

오늘 변환한 2진수는 `10111`, `11010`, `10000001`, `1111`이다. 짧은 2진수는 오른쪽 끝을 `1`의 자리와 맞추고, 왼쪽은 `0`으로 채워서 생각한다.

| 2진수 | 256 | 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 | 계산 | 10진수 |
| :--- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | :--- | ---: |
| `10111` | 0 | 0 | 0 | 0 | 1 | 0 | 1 | 1 | 1 | `16 + 4 + 2 + 1` | 23 |
| `11010` | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 1 | 0 | `16 + 8 + 2` | 26 |
| `10000001` | 0 | 1 | 0 | 0 | 0 | 0 | 0 | 0 | 1 | `128 + 1` | 129 |
| `1111` | 0 | 0 | 0 | 0 | 0 | 1 | 1 | 1 | 1 | `8 + 4 + 2 + 1` | 15 |

`10111`을 예로 들면, 오른쪽에서부터 `1`, `2`, `4`, `8`, `16` 자리다. bit가 `1`인 자리는 `16`, `4`, `2`, `1`이므로 모두 더해 `23`이 된다.

```text
10111_2 = 16 + 4 + 2 + 1 = 23_10
```

## 10진수에서 2진수로 변환

10진수를 2진수로 바꿀 때는 큰 자리수부터 사용할 수 있는지 확인한다. 해당 자리수 값을 뺄 수 있으면 bit를 `1`로 두고, 뺄 수 없으면 bit를 `0`으로 둔다.

오늘 변환한 10진수는 `14`, `25`, `59`, `279`이다.

| 10진수 | 256 | 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 | 계산 | 2진수 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | :--- | :--- |
| 14 | 0 | 0 | 0 | 0 | 0 | 1 | 1 | 1 | 0 | `8 + 4 + 2` | `1110` |
| 25 | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 0 | 1 | `16 + 8 + 1` | `11001` |
| 59 | 0 | 0 | 0 | 1 | 1 | 1 | 0 | 1 | 1 | `32 + 16 + 8 + 2 + 1` | `111011` |
| 279 | 1 | 0 | 0 | 0 | 1 | 0 | 1 | 1 | 1 | `256 + 16 + 4 + 2 + 1` | `100010111` |

`279`는 오늘 표에서 사용한 가장 큰 자리수인 `256`을 먼저 사용할 수 있다. `279 - 256 = 23`이고, 남은 `23`은 `16 + 4 + 2 + 1`로 표현된다. 따라서 자리수 bit는 `256`, `16`, `4`, `2`, `1` 위치가 `1`이 된다.

```text
279_10 = 256 + 16 + 4 + 2 + 1 = 100010111_2
```

## 포인터 기본

포인터는 변수의 값을 직접 저장하는 것이 아니라, 다른 변수의 memory 주소를 저장하는 변수다. 일반 변수 `a`가 있고 `&a`가 `a`의 주소값이라면, `int *p = &a;`는 `p`라는 포인터 변수에 `a`의 주소를 저장한다는 뜻이다.

```c
int a = 10;
int *p = &a;
```

이때 `p`와 `&a`는 같은 주소값을 나타내고, `*p`는 `p`가 가리키는 주소에 들어 있는 값을 뜻한다. 따라서 `p`를 출력하면 `&a`와 같은 주소가 나오고, `*p`를 출력하면 `a` 안에 저장된 값인 `10`이 나온다.

| 표현 | 의미 | 예시 결과 |
| :--- | :--- | :--- |
| `a` | 변수 `a`에 저장된 값 | `10` |
| `&a` | 변수 `a`가 저장된 memory 주소 | `0x...` |
| `p` | `p`에 저장된 주소값, 즉 `&a` | `0x...` |
| `*p` | `p`가 가리키는 주소의 값, 즉 `a` | `10` |
| `&p` | 포인터 변수 `p` 자체가 저장된 주소 | `0x...` |

주소 출력에는 `%p` format을 사용하고, 표준 C에서는 pointer를 `void *`로 casting해서 출력하는 형태를 쓴다.

```c
#include <stdio.h>

int main(void)
{
    int a = 10;
    int *p = &a;

    printf("a  = %d\n", a);
    printf("&a = %p\n", (void *)&a);
    printf("p  = %p\n", (void *)p);
    printf("*p = %d\n", *p);

    return 0;
}
```

출력 예시는 다음과 같은 관계를 가진다. 실제 주소값은 실행 환경마다 달라진다.

```text
a  = 10
&a = 0x16fdff2ac
p  = 0x16fdff2ac
*p = 10
```

`*` 기호는 위치에 따라 의미가 달라진다. `int *p`에서는 `p`가 `int`를 가리키는 포인터라는 선언 의미이고, `*p`에서는 `p`가 가리키는 곳의 값을 읽거나 쓰는 역참조 의미다.

### 함수 인자와 주소 전달

C 함수의 일반 인자 전달은 값 복사 방식이다. `main()`의 지역 변수 `a`를 `f1(a)`처럼 넘기면, `f1()` 안에는 이름이 같은 `a`가 있더라도 별도의 stack frame에 새 지역 변수가 만들어진다. 함수 안에서 그 값을 바꿔도 `main()` 쪽 `a`가 바뀌는 것은 아니다.

```c
void f1(int a)
{
    a = 100;
}

int main(void)
{
    int a = 10;

    f1(a);
    printf("%d\n", a);   // 10

    return 0;
}
```

`main()`의 변수를 함수 안에서 직접 바꾸려면 값이 아니라 주소를 넘겨야 한다. 함수가 `a`의 주소를 받으면, 그 주소에 `*`를 붙여 실제 `main()` 쪽 변수 공간에 접근할 수 있다.

```c
void f2(int *p)
{
    *p = 100;
}

int main(void)
{
    int a = 10;

    f2(&a);
    printf("%d\n", a);   // 100

    return 0;
}
```

이 구조 때문에 `scanf()`에서도 변수 앞에 `&`를 붙인다. `scanf()`는 함수이므로 호출한 쪽의 지역 변수를 이름으로 직접 알 수 없다. 대신 변수의 주소를 넘겨받고, 그 주소를 통해 입력값을 변수 공간에 써 넣는다.

```c
int n;

scanf("%d", &n);
```

| 전달 방식 | 함수가 받는 값 | 함수 내부 수정 결과 |
| :--- | :--- | :--- |
| `f1(a)` | `a`의 값 복사본 | 호출한 쪽 `a` 유지 |
| `f2(&a)` | `a`의 주소 | `*p`를 통해 호출한 쪽 `a` 수정 |
| `scanf("%d", &n)` | `n`의 주소 | 입력값을 `n`에 저장 |

### 문자열 배열과 문자열 포인터

문자열 literal은 프로그램 이미지 안의 정적 저장 영역에 놓이는 문자 배열이며, 수정 대상으로 취급하지 않는다. 같은 `"Hello"`를 사용하더라도 배열에 초기화하는 경우와 포인터 변수에 주소를 저장하는 경우는 memory 사용 방식이 다르다.

```c
char a[] = "Hello";
char *p = "Hello";
```

`char a[] = "Hello";`는 배열 공간을 새로 만들고, 문자열의 각 문자를 그 배열에 복사한다. `"Hello"`는 문자 5개와 마지막 null character `'\0'`까지 필요하므로 배열 크기는 6칸이 된다. 이 배열은 실제로 저장 공간을 갖고 있으므로 `a[0] = 'h';`처럼 요소 수정이 가능하다.

반면 `char *p = "Hello";`는 포인터 변수 `p`에 문자열 literal이 놓인 시작 주소를 저장한다. 이때 `p` 자체는 주소를 담는 변수이고, `p`가 가리키는 문자열 literal 영역은 수정 대상으로 보지 않는다. 실무 코드에서는 의도를 분명히 하기 위해 `const char *p = "Hello";`처럼 적는 편이 안전하다.

| 표현 | memory 형태 | 수정 가능성 |
| :--- | :--- | :--- |
| `char a[] = "Hello";` | 배열 공간에 문자 복사 | `a[i]` 수정 가능 |
| `char *p = "Hello";` | 문자열 literal 시작 주소 저장 | literal 수정 금지 |
| `const char *p = "Hello";` | 읽기 전용 문자열 주소 표현 | 수정 금지 의도 명시 |

`%s`는 문자열의 시작 주소를 받아서, 그 주소부터 한 byte씩 읽다가 null character `'\0'`을 만날 때까지 출력한다. 그래서 배열 이름 `a`도 `%s`에 넘길 수 있고, 문자열 literal `"Hello"`나 포인터 변수 `p`도 `%s`에 넘길 수 있다.

```c
printf("%s\n", a);
printf("%s\n", p);
printf("%s\n", "Hello");
```

### 포인터 연산과 배열식 접근

포인터에 정수를 더하면 단순히 주소값에 그 정수를 더하는 것이 아니라, 포인터가 가리키는 타입의 크기만큼 이동한다. `char *`에 `1`을 더하면 1 byte 이동하고, `int *`에 `1`을 더하면 보통 4 byte 이동한다.

```c
int a[4] = {10, 20, 30, 40};
int *p = &a[0];

printf("%d\n", *p);        // 10
printf("%d\n", *(p + 1));  // 20
printf("%d\n", p[1]);      // 20
```

`p[1]`은 내부적으로 `*(p + 1)`과 같은 의미로 해석할 수 있다. 포인터 변수를 배열처럼 `p[i]`로 접근할 수 있는 이유가 여기에 있다. 다만 포인터 변수가 배열 자체가 되는 것은 아니다. 포인터 변수는 주소를 저장하는 별도 object이고, 배열 이름은 많은 expression에서 첫 요소 주소로 decay되지만 주소를 다른 곳으로 바꿔 저장할 수 있는 변수는 아니다.

| 표현 | 의미 |
| :--- | :--- |
| `p` | 현재 가리키는 요소의 주소 |
| `*p` | 현재 가리키는 요소의 값 |
| `p + 1` | 다음 요소 주소 |
| `*(p + 1)` | 다음 요소 값 |
| `p[1]` | `*(p + 1)`과 같은 접근 |

포인터 변수는 주소를 저장하는 변수이므로 `p++`, `p--`처럼 증가/감소 연산도 가능하다. 이때 증가 폭은 포인터 타입의 크기 기준이다. `++p`와 `p++`는 일반 변수에서처럼 전위/후위 차이가 있으므로, `*p++`, `*++p`, `(*p)++`처럼 `*`와 함께 쓰면 평가 순서를 정확히 구분해야 한다.

포인터 type은 주소값 자체의 크기만 정하는 정보가 아니다. 같은 주소를 `char *`로 역참조하면 1byte를 읽고, `int *`로 역참조하면 보통 4byte를 읽는다. 따라서 memory-mapped register나 통신 packet을 포인터로 해석할 때는 접근 폭과 alignment가 실제 하드웨어 정의와 맞아야 한다.

## Endian과 memory 저장 순서

`MSB`/`LSB`와 endian은 둘 다 '앞뒤'를 다루는 것처럼 보여 헷갈리기 쉽지만, 기준이 다르다. `MSB`와 `LSB`는 데이터 자체 안에서 어느 쪽 byte 또는 bit가 더 큰 자리인지를 나타내고, endian은 그 데이터를 memory 주소에 어떤 순서로 저장할지를 나타낸다.

| 구분 | 의미 | 기준 |
| :--- | :--- | :--- |
| `MSB` | `Most Significant Bit/Byte`, 가장 큰 자리 쪽 | 데이터 내부의 자리 가치 |
| `LSB` | `Least Significant Bit/Byte`, 가장 작은 자리 쪽 | 데이터 내부의 자리 가치 |
| Big endian | `MSB`부터 낮은 memory 주소에 저장 | memory 저장 순서 |
| Little endian | `LSB`부터 낮은 memory 주소에 저장 | memory 저장 순서 |

예를 들어 `0x12345678`은 byte 단위로 보면 `0x12`, `0x34`, `0x56`, `0x78` 네 개로 나뉜다. 여기서 `0x12`는 가장 큰 자리 byte인 `MSB`이고, `0x78`은 가장 작은 자리 byte인 `LSB`다. 이 값 자체의 `MSB`와 `LSB`는 endian이 달라져도 바뀌지 않는다.

```text
값: 0x12345678

MSB byte: 0x12
LSB byte: 0x78
```

Endian은 이 byte들을 memory의 낮은 주소부터 어떤 순서로 놓을지 결정한다. 시작 주소를 `0x1000`이라고 하면 저장 예시는 다음과 같다.

| 구분 | Big endian | Little endian |
| :--- | :--- | :--- |
| 저장 기준 | `MSB` 먼저 | `LSB` 먼저 |
| 예시 값 | `0x12345678` | `0x12345678` |
| 낮은 주소부터 저장 | `12 34 56 78` | `78 56 34 12` |
| 읽기 편의 | 사람이 쓰는 숫자 순서와 유사 | 낮은 자리 byte부터 접근 |
| 대표 사용 | network byte order, JVM 등 | Intel/AMD x86 계열 CPU 등 |

| Address | Big endian 저장값 | Little endian 저장값 |
| :--- | :---: | :---: |
| `0x1000` | `0x12` (`MSB`) | `0x78` (`LSB`) |
| `0x1001` | `0x34` | `0x56` |
| `0x1002` | `0x56` | `0x34` |
| `0x1003` | `0x78` (`LSB`) | `0x12` (`MSB`) |

```text
Big endian memory:
낮은 주소 -> 12 34 56 78 <- 높은 주소

Little endian memory:
낮은 주소 -> 78 56 34 12 <- 높은 주소
```

정리하면 `MSB`/`LSB`는 숫자 안에서 '큰 자리'와 '작은 자리'를 부르는 이름이고, endian은 그 byte들을 memory 칸에 넣을 때 큰 자리부터 넣을지 작은 자리부터 넣을지 정하는 규칙이다.

Endian은 bit 자체를 거꾸로 저장한다는 뜻이 아니다. `0x12`라는 한 byte 내부의 bit 표현은 그대로 두고, 여러 byte를 memory 주소에 배치하는 순서가 달라지는 개념이다. 따라서 구조체, `union`, pointer casting, 통신 packet, binary file format을 볼 때 endian을 확인해야 한다.

## Union과 같은 memory 해석

`union`은 여러 member가 같은 memory 공간을 공유하는 C 문법이다. `struct`는 member마다 별도 공간을 잡지만, `union`은 모든 member가 같은 시작 주소를 사용한다. 그래서 전체 크기는 member 크기의 합이 아니라 가장 큰 member 하나를 담을 수 있는 크기로 결정된다.

| 구분 | `struct` | `union` |
| :--- | :--- | :--- |
| memory 배치 | member별 공간 분리 | 모든 member가 같은 공간 공유 |
| 전체 크기 | member 크기와 padding 합 | 가장 큰 member 기준 |
| 값 유지 | 여러 member 값을 동시에 보관 | 마지막으로 쓴 member 기준 |
| 주 사용 | 여러 값을 한 객체로 묶음 | 하나의 memory를 여러 형태로 해석 |

기본 구조는 다음과 같다.

```c
union Score {
    char grade;
    int point;
};
```

위 공용체는 `char`와 `int`를 동시에 보관하지 않는다. `point`에 값을 쓰면 같은 memory를 `grade`로 읽을 수는 있지만, 그 값은 기존 `grade` 값이 아니라 `int`의 일부 byte를 `char`로 해석한 결과가 된다.

```c
union Score score;

score.point = 95;
printf("%d\n", score.point);
```

공용체는 한 번에 하나의 의미 있는 값을 선택해서 쓸 때 memory를 줄이는 용도로 사용할 수 있다. 입력 data가 어떤 순간에는 정수이고, 다른 순간에는 실수나 문자 배열일 수 있다면 `type`이나 `flag` 같은 상태값과 함께 두어 현재 어떤 member가 유효한지 표시해야 한다.

```c
typedef enum {
    DATA_INT,
    DATA_FLOAT,
    DATA_TEXT
} DataType;

typedef struct {
    DataType type;
    union {
        int i;
        float f;
        char text[16];
    } value;
} VariantData;
```

임베디드에서는 하드웨어 register나 통신 data를 다른 단위로 해석할 때 `union`을 자주 사용한다. 예를 들어 32bit 값을 한 번에 읽은 뒤 같은 memory를 1byte 배열로 다시 해석하면, 통신 packet으로 보낼 byte 순서나 register field를 확인할 수 있다.

```c
typedef union {
    uint32_t word;
    uint8_t byte[4];
} WordBytes;
```

이 방식은 register dump, UART/I2C/SPI packet parsing, sensor data 조합에서 유용하다. 다만 `byte[0]`에 어떤 byte가 들어가는지는 CPU의 endian에 따라 달라진다. Little endian에서는 `word = 0x12345678`일 때 `byte[0]`이 `0x78`이고, big endian에서는 `byte[0]`이 `0x12`가 된다.

```c
typedef union {
    uint32_t value;
    uint8_t bytes[4];
} Data;

Data d;
d.value = 0x12345678;
```

위 코드에서 `bytes[0]`은 배열의 첫 번째 원소이면서 가장 낮은 memory 주소에 놓인 byte를 의미한다. 따라서 같은 `0x12345678`을 넣어도 endian에 따라 보이는 byte 순서가 달라진다.

| System | `bytes[0]` 낮은 주소 | `bytes[1]` | `bytes[2]` | `bytes[3]` 높은 주소 |
| :--- | :---: | :---: | :---: | :---: |
| Little endian | `0x78` | `0x56` | `0x34` | `0x12` |
| Big endian | `0x12` | `0x34` | `0x56` | `0x78` |

이처럼 한 member로 값을 쓰고 다른 member로 같은 memory를 읽는 방식을 type punning이라고 부른다. C에서는 register dump나 protocol parsing에서 흔히 쓰이지만, 모든 compiler와 모든 type 조합에서 완전히 같은 의미로 동작한다고 가정하면 안 된다. 특히 C++에서는 inactive union member 읽기가 표준적으로 더 엄격하게 다뤄지므로, 현대 C++에서는 `std::bit_cast` 같은 방식을 검토한다.

Endian 확인용 예시는 다음과 같다.

```c
int is_little_endian(void)
{
    union {
        int i;
        char c[sizeof(int)];
    } test;

    test.i = 1;
    return test.c[0] == 1;
}
```

`int` 값 `1`은 가장 낮은 byte만 `0x01`이다. Little endian이면 낮은 주소인 `c[0]`에 `1`이 들어가고, big endian이면 높은 주소 쪽에 `1`이 들어간다.

통신이나 file format처럼 byte order가 정해져 있는 data를 다룰 때는 byte 순서를 바꾸는 함수가 필요할 수 있다.

```c
uint32_t swap_endian_32(uint32_t value)
{
    union {
        uint32_t word;
        uint8_t byte[4];
    } src, dst;

    src.word = value;

    dst.byte[0] = src.byte[3];
    dst.byte[1] = src.byte[2];
    dst.byte[2] = src.byte[1];
    dst.byte[3] = src.byte[0];

    return dst.word;
}
```

실무 코드에서는 endian 변환을 직접 작성하기보다 platform에서 제공하는 `htonl()`, `ntohl()` 같은 network byte order 변환 함수나 compiler builtin을 사용할 수 있는지도 함께 확인한다.

Bit 단위 상태를 다룰 때는 `union`과 bit-field `struct`를 함께 쓰기도 한다.

```c
typedef union {
    uint8_t all;
    struct {
        uint8_t bit0 : 1;
        uint8_t bit1 : 1;
        uint8_t bit2 : 1;
        uint8_t rest : 5;
    } bits;
} PortStatus;
```

이 구조는 1byte 전체 값과 개별 bit 이름을 함께 제공하므로 register 상태 확인에 편하다. 하지만 bit-field의 실제 bit 배치는 compiler와 target에 따라 차이가 날 수 있으므로, 이식성이 필요한 register 정의에서는 data sheet와 compiler 동작을 반드시 확인해야 한다.

## C 코드와 연결

C에서 정수를 출력할 때 format 지시자를 바꾸면 같은 저장값을 다른 진법 표기로 출력한다. 표준 `printf()`는 10진수, 8진수, 16진수 출력은 직접 지원하지만, 2진수 출력 format은 표준으로 제공하지 않는다.

| format | 출력 기준 |
| :--- | :--- |
| `%d` | signed 10진수 |
| `%u` | unsigned 10진수 |
| `%o` | 8진수 |
| `%x` | 16진수 소문자 |
| `%X` | 16진수 대문자 |

2진수 값 자체를 확인하려면 직접 bit를 검사해서 출력하거나, debugger/register view에서 binary 표시를 사용하는 방식으로 본다.
