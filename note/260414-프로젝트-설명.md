# 26-04-14 프로젝트 설명 - 10000진 Up/Down Counter

## 한눈에 보기

| 항목 | 내용 |
| --- | --- |
| 목표 | Basys3에서 `0 ~ 9999` 카운터를 up/down으로 돌리고 FND에 표시한다 |
| 구조 | `button_debounce -> control_unit -> project_datapath -> fnd_controller` |
| 포인트 | 버튼 입력을 바로 카운터에 넣지 않고 제어와 데이터 경로를 분리한다 |

## 관련 코드 위치

| 파일 | 위치 | 의미 |
| --- | --- | --- |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000.v` | `19-27` | top 내부 주요 wire |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000.v` | `29-63` | 버튼 입력 conditioning |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000.v` | `65-98` | control / datapath / FND 연결 |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/control_unit.v` | `42-68` | `STOP`, `RUN`, `CLEAR`, `MODE` |
| `helloHDL/260408_counter_10000/counter_10000.srcs/sources_1/new/counter_10000_common.v` | `52-68` | up/down wrap |

## 시스템 흐름

```text
btnL, btnR, btnD
-> button_debounce
-> clear, run_stop, mode tick
-> control_unit
-> datapath
-> tick_counter
-> fnd_controller
-> FND
```

## 입력 / 출력 정리

| 신호 | 역할 |
| --- | --- |
| `btnL` | clear |
| `btnR` | run/stop |
| `btnD` | mode 전환 |
| `fnd_com`, `fnd_data` | 4자리 FND 출력 |

## 프로젝트를 설명할 때 핵심

| 항목 | 정리 |
| --- | --- |
| `button_debounce` | 버튼을 pulse로 바꿔 FSM이 읽기 쉽게 만든다 |
| `control_unit` | 지금 run인지 stop인지, clear인지, mode toggle인지 결정한다 |
| `project_datapath` | 실제 숫자를 올리거나 내린다 |
| `fnd_controller` | 값을 사람 눈에 보이는 4자리 형태로 바꾼다 |

## 주의점

| 실수 | 정리 |
| --- | --- |
| mode를 카운터 내부 flag 정도로만 본다 | 제어 정책 자체를 바꾸는 입력이다 |
| 버튼을 level로 오래 넣는다 | 이 프로젝트는 1클럭 tick 입력으로 읽는 편이 안정적이다 |
| control과 datapath를 한 파일에서 뒤섞는다 | 설명과 디버깅이 어려워진다 |
