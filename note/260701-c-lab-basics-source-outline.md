## 1. 출력, 상수, 변수, 문자

### 1.1 C 프로그램의 기본 형태

`[1-1]`은 `main()` 함수와 `printf()` 출력 흐름을 확인하는 첫 예제다. `main()`은 C 프로그램의 시작 지점이고, `return 0;`은 프로그램이 정상 종료되었음을 운영체제에 돌려주는 값으로 본다.

```c
#include <stdio.h>

int main(void)
{
    printf("Hello C World!\n");
    return 0;
}
```

`stdio.h`는 standard input/output 관련 선언을 담고 있으므로 `printf()`나 `scanf()`를 사용할 때 포함한다. `printf()`는 format string을 해석해 화면에 출력하고, 문자열 안의 `\n`, `\t` 같은 escape sequence도 처리한다.

### 1.2 여러 줄 출력과 escape sequence

`[1-2]`부터 `[1-4]`는 `printf()`를 여러 번 호출하거나 문자열 안에서 특수 문자를 표현하는 방법을 다룬다.

| 표기 | 의미 | 사용 |
| :--- | :--- | :--- |
| `\n` | newline | 다음 줄로 이동 |
| `\t` | horizontal tab | tab 간격 삽입 |
| `\\` | backslash 문자 | `\` 자체 출력 |
| `\"` | double quote 문자 | 문자열 안에서 `"` 출력 |
| `\'` | single quote 문자 | 문자/문자열 안에서 `'` 출력 |
| `\0` | null character | C 문자열 종료 표지 |

escape sequence는 문자 하나처럼 동작하지만, source code에서는 backslash와 뒤 문자 조합으로 표현된다. 예를 들어 `\\`는 화면에 backslash 하나를 출력하기 위한 표현이다.

### 1.3 상수와 format 지시자

`[1-5]`는 정수, 실수, 문자, 문자열 상수를 format 지시자와 연결한다. format 지시자는 출력할 값의 해석 방식을 `printf()`에 알려 주는 약속이다.

| 값 종류 | 예 | format |
| :--- | :--- | :--- |
| 정수 | `10` | `%d` |
| 실수 | `3.14` | `%f` |
| 문자 | `'A'` | `%c` |
| 문자열 | `"Hello"` | `%s` |

format과 실제 인자의 타입이 어긋나면 출력이 깨지거나 정의되지 않은 동작이 발생할 수 있다. 초반 실습에서는 format 지시자와 인자 순서를 정확히 맞추는 것이 중요하다.

### 1.4 변수 선언, 정의, 초기화

`[1-6]`은 변수의 기본 형식을 확인한다. 변수는 값을 저장할 이름 있는 공간이고, 타입은 그 공간의 크기와 해석 방식을 결정한다.

```c
int count = 10;
double ratio = 3.14;
char grade = 'A';
```

| 용어 | 예 | 의미 |
| :--- | :--- | :--- |
| 선언/정의 | `int count;` | `int` 값을 저장할 공간 준비 |
| 초기화 | `int count = 10;` | 변수 생성과 동시에 첫 값 저장 |
| 대입 | `count = 20;` | 이미 있는 변수의 값 변경 |

변수 이름은 숫자로 시작할 수 없고, 공백을 포함할 수 없으며, C keyword와 겹치면 안 된다. C는 대소문자를 구분하므로 `abc1`과 `Abc1`은 서로 다른 이름이다.

### 1.5 문자는 숫자다

`[1-7]`은 `char`와 ASCII code의 관계를 다룬다. `char`는 문자 전용 타입처럼 보이지만 실제로는 작은 정수형 타입이고, 문자 literal은 내부적으로 숫자 값으로 저장된다.

```c
char ch = 'A';

printf("%c\n", ch);
printf("%d\n", ch);
```

같은 `ch`를 `%c`로 출력하면 문자 `A`로 보이고, `%d`로 출력하면 ASCII code 값인 `65`로 보인다. 출력 format이 값을 문자로 해석할지 정수로 해석할지를 결정한다.

소문자와 대문자 사이에는 ASCII code 기준으로 32 차이가 난다. 예를 들어 `'q' - 32`는 `'Q'`에 해당한다. 다만 이 방식은 ASCII 전제에 의존하므로 실제 문자열 처리에서는 표준 함수 사용 여부도 함께 고려해야 한다.

## 2. 입력

### 2.1 `scanf()`와 주소 전달

`[2-1]`은 `scanf()`로 입력값을 변수에 저장하는 예제다. `scanf()`는 호출한 쪽의 지역 변수를 이름으로 직접 알 수 없으므로, 값을 저장할 변수의 주소를 argument로 받아야 한다. 그래서 일반 변수 앞에는 address-of 연산자 `&`를 붙인다.

```c
int n;

scanf("%d", &n);
printf("%d\n", n);
```

`&n`은 변수 `n`의 주소를 의미한다. `scanf()`는 입력된 문자열을 format에 맞게 변환한 뒤, 그 주소에 결과를 써 넣는다.

### 2.2 여러 값 입력

`[2-2-1]`, `[2-2-2]`는 여러 개의 정수를 입력받는 방식이다. 하나씩 `scanf()`를 여러 번 호출할 수도 있고, 하나의 format string 안에 여러 format 지시자를 넣어 한 번에 받을 수도 있다.

```c
int a, b, c;

scanf("%d %d %d", &a, &b, &c);
```

공백, tab, newline은 정수 입력에서 구분자처럼 처리된다. 따라서 사용자가 한 줄에 입력하든 여러 줄에 나눠 입력하든 format과 값 개수가 맞으면 같은 결과가 된다.

### 2.3 문자 입력과 입력 buffer

`[2-3-1]`부터 `[2-4-2]`는 문자 입력에서 남아 있는 newline이 영향을 줄 수 있음을 확인한다. 정수 입력 뒤에 문자 입력을 바로 받으면, 이전 입력에서 남은 newline이 문자로 읽힐 수 있다.

```c
int n;
char ch;

scanf("%d", &n);
scanf(" %c", &ch);
```

`" %c"`처럼 `%c` 앞에 공백을 넣으면 이전 공백 문자와 newline을 건너뛰고 실제 문자를 읽도록 만들 수 있다. 문자 입력에서는 입력 buffer의 남은 문자를 의식해야 한다.

### 2.4 문자열 입력

`[2-5]`는 `%s`로 문자열을 입력받는 예제다. 문자열을 저장할 배열은 충분한 크기로 잡아야 하고, `%s`는 공백 전까지만 읽는다.

```c
char name[32];

scanf("%s", name);
```

문자 배열 이름 `name`은 배열의 시작 주소처럼 전달되므로 `&name`이 아니라 `name`을 넘긴다. 문자열 뒤에는 null character인 `'\0'`이 들어가야 하므로 실제 입력 가능한 글자 수는 배열 크기보다 1 작게 생각해야 한다.

`[2-6]`은 공백이 포함된 문자열 입력을 다룬다. `%s`는 공백에서 끊기므로 문장 단위 입력이 필요할 때는 별도 입력 방식이나 scanset 같은 형식이 필요하다.

## 3. 계산

### 3.1 사칙연산과 우선순위

`[3-1]`, `[3-2]`는 C의 기본 산술 연산자와 우선순위를 확인한다.

| 연산자 | 의미 |
| :--- | :--- |
| `+` | 덧셈 |
| `-` | 뺄셈 |
| `*` | 곱셈 |
| `/` | 나눗셈 |
| `%` | 나머지 |

곱셈, 나눗셈, 나머지 연산은 덧셈과 뺄셈보다 우선순위가 높다. 의도를 분명히 하려면 괄호를 사용하는 것이 좋다.

```c
int a = 10 + 20 / 2;     // 20
int b = (10 + 20) / 2;   // 15
```

### 3.2 `/`와 `%`

`[3-3]`은 몫과 나머지를 분리하는 예제다. 정수끼리 `/`를 사용하면 정수 몫이 나오고, `%`는 나머지를 구한다.

```c
int n = 1234;

int d1 = n / 1000;
int d2 = n / 100 % 10;
int d3 = n / 10 % 10;
int d4 = n % 10;
```

자리수 분리, 주기 계산, 홀짝 판단, 특정 범위 순환 같은 문제에서 `/`와 `%`가 자주 사용된다.

### 3.3 복합대입과 증감 연산

`[3-4]`, `[3-5]`는 값을 누적하거나 1씩 변화시키는 표현을 다룬다.

| 표현 | 같은 의미 |
| :--- | :--- |
| `a += b` | `a = a + b` |
| `a -= b` | `a = a - b` |
| `a *= b` | `a = a * b` |
| `a /= b` | `a = a / b` |
| `a++` | 현재 식 평가 후 1 증가 |
| `++a` | 먼저 1 증가 후 식 평가 |

`a++`와 `++a`는 단독 문장에서는 결과가 같아 보이지만, 다른 연산과 함께 쓰면 평가 순서 차이가 드러난다. 초반에는 증감 연산자를 복잡한 식 안에 섞기보다 별도 문장으로 분리하는 편이 안전하다.

### 3.4 cast, `sizeof`, `&`

`[3-6]`은 명시적 타입 변환을 다룬다. 정수 나눗셈 결과를 실수로 보고 싶으면 피연산자 중 하나를 실수형으로 바꿔야 한다.

```c
int a = 10;
int b = 3;

printf("%f\n", (double)a / b);
```

`[3-7]`은 `sizeof`와 주소 연산자 `&`를 확인한다. `sizeof`는 타입이나 객체의 byte 크기를 구하고, `&`는 변수의 주소를 구한다.

```c
printf("%zu\n", sizeof(int));
printf("%p\n", (void *)&a);
```

주소는 이후 `scanf()`, pointer, 배열, 구조체를 이해하는 기반이 된다.

## 4. 나만의 함수 만들기

### 4.1 함수의 기본 구성

`[4-1]`부터 `[4-4]`는 함수가 입력(parameter)과 출력(return)을 가질 수 있음을 확인한다.

```c
int add(int a, int b)
{
    return a + b;
}
```

| 형태 | 의미 | 예 |
| :--- | :--- | :--- |
| parameter 없음, return 없음 | 동작만 수행 | `void hello(void)` |
| parameter 있음, return 없음 | 입력값으로 동작 | `void print_num(int n)` |
| parameter 없음, return 있음 | 내부 결과 반환 | `int get_value(void)` |
| parameter 있음, return 있음 | 입력으로 결과 계산 | `int add(int a, int b)` |

함수는 반복되는 code를 이름 있는 기능 단위로 분리한다. `main()`은 전체 실행 순서를 담당하고, 개별 함수는 한 가지 역할에 집중하도록 구성한다.

### 4.2 함수의 위치와 선언

`[4-5]`, `[4-6]`은 함수가 호출되기 전에 컴파일러가 그 함수의 형태를 알고 있어야 함을 보여 준다. 함수 정의가 호출보다 아래에 있다면 prototype을 먼저 선언한다.

```c
int add(int a, int b);

int main(void)
{
    printf("%d\n", add(10, 20));
    return 0;
}

int add(int a, int b)
{
    return a + b;
}
```

prototype은 함수의 이름, parameter 타입, return 타입을 알려 주는 선언이다. 실제 동작은 함수 정의에서 작성한다.

### 4.3 변수의 유효 범위

`[4-7]`은 scope를 다룬다. 지역 변수는 선언된 block 안에서만 사용할 수 있고, 전역 변수는 파일 전체에서 접근할 수 있다.

| 변수 종류 | 선언 위치 | 특징 |
| :--- | :--- | :--- |
| 지역 변수 | 함수/block 내부 | block 종료 시 수명 종료 |
| 전역 변수 | 함수 외부 | 여러 함수에서 접근 가능 |
| parameter | 함수 선언부 | 함수 호출 시 전달값 복사 |

전역 변수는 편리하지만 의존 관계가 숨기 쉬우므로, 초반 실습에서는 함수 parameter와 return으로 데이터를 주고받는 습관이 중요하다.

### 4.4 `#define`과 macro

`[4-8]`, `[4-9]`는 macro를 다룬다. `#define`은 전처리 단계에서 단순 치환되므로, 상수 이름을 붙이거나 짧은 식을 재사용할 때 사용할 수 있다.

```c
#define ARR_SIZE 10
#define SQUARE(x) ((x) * (x))
```

함수형 macro는 괄호가 중요하다. `#define SQUARE(x) x * x`처럼 쓰면 `SQUARE(a + b)`가 `a + b * a + b`처럼 치환되어 의도와 달라질 수 있다. 인자와 전체 식에 괄호를 넣어야 한다.
