# Verilog HDL 복습노트 01 - 설계 흐름

## 한 줄 요약

Verilog 실습은 `코드 작성`보다 `검증 순서`를 먼저 잡는 것이 중요하다.

## 설계 흐름

| 단계 | 의미 | 확인할 것 |
| --- | --- | --- |
| Design Source 작성 | RTL 모듈을 만든다. | 포트, 비트폭, 조합/순차 구분 |
| Simulation Source 작성 | testbench를 만든다. | 입력 자극, 종료 조건, 파형 확인 |
| Simulation | 기능을 먼저 검증한다. | 예상 출력과 실제 출력 일치 여부 |
| Synthesis | HDL을 논리 회로로 바꾼다. | latch, 경고, 자원 사용량 |
| Implementation | FPGA 자원에 배치/배선한다. | 타이밍, 핀 연결, 제약조건 |
| Bitstream | 다운로드 파일을 만든다. | 오류 없이 생성되는지 |
| Program Device | 보드에 올린다. | 실제 입출력 동작 |

## 자주 헷갈리는 구분

| 항목 | 뜻 |
| --- | --- |
| Elaborated Design | RTL 구조를 툴이 풀어 본 중간 형태 |
| Synthesis | RTL을 게이트 수준 구조로 바꾸는 단계 |
| Implementation | 배치/배선과 타이밍까지 포함한 FPGA 구현 단계 |
| Bitstream | FPGA에 다운로드하는 최종 파일 |

## 실습 원칙

| 원칙 | 이유 |
| --- | --- |
| 시뮬레이션을 먼저 본다. | 논리 오류를 가장 싸게 찾을 수 있다. |
| XDC를 마지막까지 같이 본다. | 보드 오류와 설계 오류를 섞어 보면 원인 분리가 어렵다. |
| 작은 모듈부터 검증한다. | 상위 모듈 디버깅이 훨씬 쉬워진다. |
