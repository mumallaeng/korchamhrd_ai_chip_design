# 디지털 회로 설계 일지 템플릿 모음

부트캠프 프로젝트 일지 | 작업 유형별 확장 가이드

---

## 파일 구성

| 파일명 | 작업 유형 | 사용 시점 |
|--------|----------|----------|
| `TYPE_A_설계.md` | 설계 (RTL/아키텍처) | 블록 다이어그램, 인터페이스 정의, 마이크로아키텍처 설계일 |
| `TYPE_B_코딩.md` | 코딩 (SystemVerilog) | RTL 코드 작성, FSM 구현일 |
| `TYPE_C_검증.md` | 검증 (시뮬레이션/디버깅) | 테스트벤치 작성, 파형 분석, 디버깅일 |
| `TYPE_D_문서.md` | 문서 작업 | 설계 스펙, 검증 계획, 회의록 작성일 |
| `TYPE_E_학습.md` | 학습 / 복습 | 강의 수강, 개념 정리, 실습일 |
| `TYPE_F_발표.md` | 발표 | 발표 당일 (3번 항목이 Feedback으로 변경) |
| `TYPE_MIX_복합.md` | 복합 작업 | 2~3가지 작업 유형이 섞인 날 |

---

## 사용 규칙

1. **1)~4) 번호와 제목은 변경 불가** — 양식 고정
2. **1) 하위 `###` 소항목**은 해당일 작업 유형에 맞게 선택·조합
3. 해당 내용이 없는 항목은 `N/A`로 표기
4. 복합 작업일은 `TYPE_MIX`를 기준으로, 필요한 유형의 소항목만 포함

---

## 작업 흐름 예시

```
설계만 한 날     → TYPE_A_설계.md
코딩만 한 날     → TYPE_B_코딩.md
설계 + 코딩 날   → TYPE_MIX_복합.md  ([설계] + [코딩] 소항목 포함)
발표 당일        → TYPE_F_발표.md
발표 후 청강 날  → TYPE_E_학습.md 또는 TYPE_MIX_복합.md
```
